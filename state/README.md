# State — current snapshots

This folder holds point-in-time narrative snapshots of the user's finances. Think of it as "what's true right now, in English." Numeric truth lives in the SQLite database; this folder is the narrative layer.

## What lives here

`finance init` (run once at install time) copies seven scaffold files into this folder from `templates/state/`:

- `net-worth.md` — current net worth narrative. Regenerated by the monthly routine from `finance net-worth --json`.
- `income.md` — all income sources, cadence, taxability notes.
- `debts.md` — every debt with balance, APR, minimum payment, payoff plan.
- `insurance.md` — coverage types, carriers, premiums, review dates.
- `tax.md` — filing status, state, major deductions, quarterly estimates if self-employed.
- `estate.md` — will status, beneficiaries, POA, healthcare directive.

If any of those files are missing on your machine, run `finance init` again — it's idempotent and will only copy files that don't already exist.

## Hand-maintained vs. regenerated

- **Regenerated each month** from the DB: `net-worth.md`.
- **Hand-maintained** (the advisor edits them with your confirmation): the other five.

## Staleness

Each file has a frontmatter `updated` date and a `stale_after` interval. The advisor flags stale state before using it. For insurance and estate, stale typically means "re-review with a professional, not just re-edit the file."

## What doesn't live here

- Transaction-level data — that's in the database.
- Long-term plans — those live in `STRATEGY.md`.
- Stated preferences — those live in `memory/preferences/`.
- Full account narratives — those live in `accounts/`.
